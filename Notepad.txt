第一部分 Java 入门：

【必备基础知识细节/非细节】：

1)Java语法规定：源程序文件名必须与类名相同，否则在编译的时候会报错。

2)Java的方法必须存在于某个类当中，方法是不能独立存在的。

3)main()方法是类中必须有的，是所有Java程序的运行起点，如果没有main()方法，编译的时候不会报错。但是运行时，会因找不到main()方法而抛出错误。

4)名词解释：
  SDK(Software Development Kit)。
  JDK(Java Development Kit) 是 Java 语言的软件开发工具包(SDK)。
  SE(JavaSE)，standard edition，标准版，是我们通常用的一个版本，从JDK 5.0开始，改名为Java SE。
  EE(JavaEE)，enterprise edition，企业版，使用这种JDK开发J2EE应用程序，从JDK5.0开始，改名为JavaEE。从2018年2月26日开始，
J2EE改名为Jakarta EE [1-2]  。
  ME(J2ME)，micro edition，主要用于移动设备、嵌入式设备上的java应用程序，从JDK 5.0开始，改名为Java ME。
没有JDK的话，无法编译Java程序（指java源码.java文件），如果想只运行Java程序（指class或jar或其它归档文件），要确保已安装相应的JRE。
  J2EE(Java 2 Platform, Enterprise Edition)是一个为大企业主机级的计算类型而设计的Java平台。Sun微系统（与其工业伙伴一起，例
如IBM）设计了J2EE，以此来简化在受客户级环境下的应用开发。由于创造了标准的可重用模块组件以及由于构建出能自动处理编程中多方面问题的等级结构，J2EE简化了应用程序的开发，也降低了对编程和对受训的程序员的要求。

5)
Java 编译器（Java Compiler），就是 javac xxx 命令。
Java 类库（Java Class Libraries），万物之源。
Java 虚拟机（Java Virtual Machine）（JVM），就是 java xxx 命令以启动Java虚拟机，去执行一个java程序。
*.class 文件必须由命令行 java xxx 才能执行，不同于 .exe文件，此时对于 .class 文件来说，Java虚拟机就是Windows。

【Java 基本语法与规则】

1.Java 8种基本数据类型：byte、short、int、long、float、double、char、boolean;其中表示数据的类型精度按从小到大的顺序排列为
byte、short、int、long、float、double。

2.运算符/逻辑运算符/位操作符同Js，不再赘述。 
*无论是Java还是Js，逻辑运算符诸如||、&&等，前置位表达式会影响后续是否继续运算，可以此来提高性能。

3.类型强制转换： (类型)变量名。
*强制转换后，会直接改变原变量类型。
*强转相较于其他算数运算符而言优先级最高。
*小数向整数强制转换时，不会四舍五入，而是直接砍掉全部小数部分。
*强制转换有风险，比如大值域往小值域强转，如果超出范围，则会发生不可预期的错误（应该也并非完全不可预测）。
*隐转无法进行从高精度往低精度类型上转换操作，但是强转可以。
*[关于变量声明]：
a)注重的是结果变量类型的声明，声明变量类型时，注意结果变量类型应该是整个式子中所有变量类型中的最高级类型，如果不满意该类型应该换一行再强转。
b)变量一旦声明类型，之后的赋值操作中，只能把结果数据类型精度高于等于此声明类型的值赋给之，不然会报错；除非进行结果整体的强转。

4.浮点数比较问题：Java 中小数只有近似，没有绝对，只有整数才能绝对相等。

5.关于数组：
*Java 数组内的元素必须得是同样类型的数据（声明时的类型）。[不同于Js]
*Java 数组大小一旦声明则已被固定，没有办法扩容或缩小。[不同于Js]
*Java 数组引用其中元素时不能超出其范围，不然报错。[不同于Js]
*强转对于数组整体来说无能为力。当对某个数组元素进行强转赋值时，将转换之后的结果，按照从低精度往高精度可转化的特性，最终会尝试将其转换为跟声明该数组时类型相同的类型，如果不能则报错。
*数组的简单声明是{}而不像Js的[]。
*各种类型的数组声明后都有其默认初始值。
*同Js一样的指针式数组名称。
*多维数组一旦有一维是[0]，则整个数组都是空的，注意这一点。
